# DINOv1/CLIP K=11 Failure Analysis

**Date**: 2026-03-02

---

## 背景

K=5→K=11 でスロット数を増やした結果、バックボーンごとに劇的に異なる結果が出た：

| Backbone | K=5 FG-ARI | K=11 FG-ARI | 変化 |
|---|---|---|---|
| **DINOv2** | 0.155 | **0.470** | +0.315 ★ 大幅改善 |
| DINOv1 | 0.176 | 0.131 | -0.045 悪化 |
| CLIP | 0.047 | 0.110 | +0.063 微改善 |

同じアーキテクチャ（Slot Attention + 同一デコーダ）でバックボーンだけが異なる。
なぜDINOv2だけがK=11の恩恵を受けるのか？

---

## 分析プロセス

### Step 1: バックボーン特徴量の空間構造を調べる（§2）

**目的**: 各バックボーンが出力する特徴量が「空間的にどれだけ多様か」を定量化

**方法**: 20シーンの特徴量について、パッチ間のコサイン類似度・空間分散・PCA次元数を計測

**結果**:

| 指標 | DINOv2 | DINOv1 | CLIP |
|---|---|---|---|
| コサイン類似度 | 0.52 | 0.34 (最も多様) | 0.92 (ほぼ均一) |
| 空間分散 | 2.69 | 11.88 (最大) | 0.04 (最小) |
| PCA 90%次元数 | 20 | 45 (最も複雑) | 3 (最も単純) |
| ノルム | 46 | 81 | 13 |

**発見**: CLIPは全パッチがほぼ同一方向（cos=0.92）。CLIPは画像-テキスト対応を学習するモデルであり、「この画像は犬」のような画像全体のカテゴリ情報を符号化するが、「犬がどこにいるか」の空間情報はほとんど持たない。DINOv1は空間的に最も多様だが情報が45次元に分散。

### Step 2: 投影（384→64次元）で何が起きるか調べる（§3）

**目的**: Slot Attentionは64次元で動作するため、バックボーンの384次元特徴量をMLPで圧縮する。この圧縮で空間情報が保たれるか検証

**投影（Projection）とは**: `LayerNorm → Linear(384,384) → ReLU → Linear(384,64)` の学習可能な変換

**結果**:

| | DINOv2 | DINOv1 | CLIP |
|---|---|---|---|
| 投影前 コサイン類似度 | 0.51 | 0.34 | 0.93 |
| 投影後 コサイン類似度 | **0.31 ↓改善** | **0.19 ↓改善** | **0.99 ↑悪化** |
| LayerNorm学習度(std) | 0.034 | 0.042 | **0.002** |

**発見**: DINOv2/DINOv1では投影が空間的多様性を増幅する。CLIPでは投影がさらに均一化を悪化させる（0.93→0.99）。CLIPの投影層はほぼ未学習（LayerNorm重みが初期値1.0からほとんど変化なし）。元の特徴量が均一すぎて学習信号自体が弱い。

### Step 3: マスクの品質を評価する（§4）

**目的**: Slot Attentionが出力するマスク（各スロットの担当領域）の質を直接評価

**結果（50シーン平均）**:

| 指標 | DINOv2 | DINOv1 | CLIP |
|---|---|---|---|
| アクティブスロット | **11.0/11** | 6.6/11 | 9.6/11 |
| トップスロット占有率 | 30% | **52%** (支配) | 18% |
| エントロピー / max | 0.60 | 0.47 | **0.88** (均一) |
| ジニ係数 | 0.33 | **0.65** | 0.36 |

**発見**: DINOv1は「スロット崩壊」— 1つのスロットが全体の52%を占拠し、残りが非活動化。CLIPはエントロピーが最大に近い（0.88）— 全マスクが均一で位置区別不能。DINOv2は全11スロットが適度に機能分化。

### Step 4: 再構成品質を「Trivial Mean-Fill」基準で判定する（§5）

**目的**: モデルが本当に空間構造を学習しているか判定

**Trivial Mean-Fill（自明な平均充填）とは**: 画像全体の特徴量の平均を全位置にコピーする最も単純な戦略。学習不要。

**判定**: `再構成MSE / Mean-Fill MSE` — < 1.0なら学習済み、≈ 1.0は未学習、> 1.0は逆効果

**結果（20シーン平均）**:

| | DINOv2 | DINOv1 | CLIP |
|---|---|---|---|
| Reconstruction MSE | 0.92 | 12.84 | 0.08 |
| Trivial Mean-Fill MSE | 2.69 | 11.88 | 0.04 |
| **Ratio** | **0.34 ✅** | **1.09 ⚠️** | **2.16 ❌** |

- **DINOv2**: 再構成はtrivialの34%。空間構造を忠実に復元
- **DINOv1**: 平均コピーと同等。199エポック訓練しても空間構造を学習できず
- **CLIP**: 平均コピーより悪い。trivial MSEが0.04と極小、epoch 4で収束

## 結論: Goldilocks特性

「Goldilocks（ゴルディロックス）」は童話「3匹のくま」で少女が「熱すぎず冷たすぎない、ちょうどいい粥」を選ぶ話に由来する比喩。

DINOv2はSlot Attentionにとって3つの意味で「ちょうどいい」：

1. **空間的多様性**: CLIPのように均一すぎず、DINOv1のように複雑すぎない
2. **情報のコンパクトさ**: 20 PCA次元で64次元スロットに収まる（DINOv1は45次元必要）
3. **特徴量スケール**: norm=46が適切（DINOv1=81は大きすぎ、CLIP=13は小さすぎ）

### 失敗メカニズム

**CLIP**: 特徴量が空間均一 → 投影で悪化 → 位置区別不能 → 全マスク均一 → epoch 4で自明解に収束

**DINOv1**: 特徴量は空間豊かだが45次元必要 → 384→64投影で情報喪失 → 1スロット支配 → 再構成≈trivial

**DINOv2**: 適度な多様性 + コンパクト構造 → 投影で多様性増幅 → 全11スロット活動 → FG-ARI 0.470
