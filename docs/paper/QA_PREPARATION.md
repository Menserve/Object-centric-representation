# ポスター発表 想定Q&A集

**発表日**: 2026-03-08  
**ポスター**: `docs/paper/slide.pdf`

---

## Q1: 「なぜ材質ラベルがランダムだったのか」

**背景**: 論文提出版で使用した材質ラベルが、実はランダム生成されたものだった。

**回答**:
- 初期の `download_movi.py` でMOVi-AのTFDSレコードから `instances/material_label` を取得せず、30%の確率でランダムに "metal" ラベルを付与していた
- Phase 2のデバッグ（スロット崩壊、正規化問題、CLIPバグ等）に追われ、データパイプラインの検証を怠った
- 論文提出後に発見し、正しいラベルで再評価を実施
- **教訓**: データパイプラインのsanity checkは最初にやるべき

## Q2: 「Lambert対照群の結果は？」

**回答**:
- MOVi-A 300シーンのうち、rubber_only（全物体がrubber）は5シーンのみ、metal_onlyは8シーン
- サンプルサイズが小さすぎて統計的に有意な結論は出せない
- per-objectでのmaterial ARIでは、DINOv2 K=11でMetal=0.491 vs Rubber=0.446（$p=0.12$）
- 方向は「MetalがRubberより良い」だが統計的に有意ではない

## Q3: 「FG-ARI 0.470は低くないか？」

**回答**:
- 元のDINOSAUR論文（Seitzer et al., 2023）のMOVi-CでのFG-ARIは0.59-0.64
- MOVi-Aは物体数が3-10と多様で、MOVi-Cの10個固定より難しい面がある
- 我々のモデルは意図的に小規模（slot_dim=64, hidden_dim=128）で、RTX 5090 1枚で数分で訓練可能
- 大規模モデル（slot_dim=128, hidden_dim=256）にすれば改善の余地はあるが、本研究の主眼はバックボーン比較

## Q4: 「Goldilocks特性は一般的に成り立つか？」

**回答**:
- 本研究はMOVi-A（合成データ、128×128）での検証のみ
- DINOv2の「適度な空間多様性 + コンパクトな情報構造」がSlot Attentionに適しているという仮説は、他のデータセットでの検証が必要
- 特に実世界画像（COCO、Pascal VOC等）では特徴量の性質が変わる可能性がある
- ただし、CLIPの空間情報欠如とDINOv1の情報分散という問題はアーキテクチャに起因するため、一般的に成り立つ可能性が高い

## Q5: 「なぜDINOv1はK=11で悪化するのか？」

**回答**: 
- DINOv1の特徴量は空間的に最も多様（cos=0.34）だが、情報が45次元に分散
- 384→64の投影で情報が失われ、Slot Attentionが11個のスロットを区別できなくなる
- 結果として1つのスロットが全体の52%を占有する「スロット崩壊」が発生
- K=5では必要な情報量が少ないため問題が表面化しにくい

## Q6: 「ch-norm損失とは何か？なぜDINOv1に必要？」

**回答**:
- channel normalization: 特徴量チャンネルごとの平均・分散で正規化してからMSEを計算
- DINOv1の特徴量はnorm=81と大きく（DINOv2=46, CLIP=13）、スケールが不均一
- 標準MSEでは高ノルムチャンネルが損失を支配してしまう
- ch-normにより全チャンネルを等しく重み付けし、空間情報の学習が可能になる

## Q7: 「CLIP detachとは何か？」

**回答**:
- CLIPのViTはopen_clipライブラリ経由で使用。デフォルトではBackwardで勾配がViTまで流れる
- 凍結ViTの場合は不要な勾配計算で、メモリ増大・学習不安定化の原因
- `.detach()`で勾配を明示的に遮断し、post-projection部分のみ学習
- これが無い場合、CLIPのFG-ARIは0.01以下まで低下

## Q8: 「16×16パッチ解像度の壁とは？」

**回答**:
- ViT-S/14は224÷14=16のパッチグリッドで画像を処理
- Slot Attentionのマスクは16×16解像度で生成され、物体境界がぼやける
- 小さい物体（数ピクセル）は1パッチ内に収まり、分離不能
- 解決策: SAM等の高解像度マスクとの後処理統合、またはhierarchical slot attention

## Q9: 「論文からポスターで変わった点は？」

**回答**:
1. **RQの一本化**: 鏡面反射+バックボーン比較 → バックボーンの事前学習目的が物体中心学習に与える影響
2. **材質ラベル修正**: ランダム → 正しいper-object materialラベルで再評価
3. **K=11実験の追加**: K=5ではDINOv2≈DINOv1だったが、K=11でDINOv2のみ大幅改善
4. **失敗要因分析の追加**: Goldilocks特性の発見とメカニズムの解明
5. **データ拡大**: 60シーン → 300シーン

## Q10: 「再構成/Mean-Fill比とは何か？」

**回答**:
- Mean-Fill: 画像全体の特徴量の平均を全位置にコピーする最も単純な戦略（学習不要）
- 「モデルの再構成MSE ÷ Mean-FillのMSE」で学習の質を判定
- <1.0: モデルが空間構造を学習 (DINOv2: 0.34)
- ≈1.0: 学習してない (DINOv1: 1.09)
- >1.0: Mean-Fillより悪い (CLIP: 2.16) — 学習が逆効果
