## 2. 発見された致命的矛盾：材質ラベルがランダムだった

### 2.1 問題の所在

`experiments/main/download_movi.py` L88:
```python
# material 分類を推定（ランダムに分ける）
is_metal_only = np.random.rand() < 0.3  # 30%をmetal_onlyとして扱う
```

MOVi-AのTFDSレコードから `sample['instances']['material_label']` を取得せず、
**30%の確率でランダムに "metal" ラベルを付与**していた。

### 2.2 影響範囲

| 影響を受ける結果 | 状況 |
|---|---|
| Table 2（材質別FG-ARI） | **完全に無効**。2つのランダム部分集合の比較に過ぎない |
| §5.2（鏡面反射によるスロット混同） | 定性分析は実画像に基づくため一部有効だが、「metal物体」との対応が不正確 |
| Abstract/Conclusion の材質関連の主張 | 根拠を失う |

| 影響を受けない結果 | 理由 |
|---|---|
| Table 1（Backbone比較 FG-ARI） | 全サンプルでの評価。ラベルに依存しない |
| 温度スケーリング、2層MLP等のアブレーション | 材質ラベルと無関係 |
| CLIP勾配漏洩バグの発見・修正 | 材質ラベルと無関係 |
| 構造的限界の定性分析（図1: にじみ） | パッチ解像度の問題。材質ラベルと無関係 |

### 2.3 なぜ気づかなかったか

- Phase 2のデバッグ（スロット崩壊→正規化→CLIPバグ）に追われ、データパイプラインの検証を怠った
- `download_movi.py` は Phase 1 初期に作成され、以降見直されなかった
- Phase 3（分析）がほぼカットされたため、結果の sanity check が不十分だった

### 2.4 救える点

MOVi-A TFDSレコードには `instances/material_label`（0=metal, 1=rubber）が**存在する**。
per-object でアクセス可能:
```python
sample['instances']['material_label'][i].numpy()  # 0=metal, 1=rubber
```
データの再生成は不要。ダウンロードスクリプトの修正と再取得のみで修復可能。

## 3. ポスター改訂の方針

### 3.1 ストーリーの再構成

**旧ストーリー**（論文提出版）:
> 鏡面反射物体は追跡が困難 → 3 backboneで検証 → DINOv2が最良 → 構造的限界を発見

**新ストーリー**（ポスター版）:
> 事前学習目的の異なるViTは物体中心学習でどう振る舞うか → DINOv2 > DINOv1 >> CLIP
> → 正しい材質ラベルで再評価し Metal/Rubber の実際の影響を測定
> → 構造的限界（パッチ解像度、スロット混同）+ 工学的知見（正規化、温度、バグ）

**RQの一本化**:
> 「異なる事前学習目的で訓練されたViTの特徴量は、物体中心学習フレームワークの中でどう振る舞い、材質（metal/rubber）による影響はどの程度か？」

### 3.2 方向A（材質検証の修復）と方向B（Backbone比較の強化）の統合

- **方向A**: ダウンロードスクリプト修正 → 正しい material_label で 300+サンプル取得 → 既存モデルで再評価
- **方向B**: Backbone比較を主軸に据え、材質分析は「正しいラベルでの追加検証」として位置づけ
- **統合**: ポスターでは Backbone比較が主結果、材質分析は「論文提出後に発見した問題を修正した結果」として追加

## 7. Day 1 実行結果 (2026-03-01)

### 7.1 データ修復完了

`experiments/main/download_movi_v2.py` を新規作成し、MOVi-A TFDS の `instances/material_label` から
正しい材質ラベルを取得。300サンプルを `data/movi_a_v2/` に保存。

**データ統計**:
| 分類 | サンプル数 | 割合 |
|------|-----------|------|
| mixed (metal+rubber混合) | 287 | 95.7% |
| metal_only | 8 | 2.7% |
| rubber_only | 5 | 1.7% |

- オブジェクト単位: metal 985個 / rubber 984個 (**ほぼ50:50**)
- 平均6.6オブジェクト/シーン (min=3, max=10)
- friction値によるcross-check: 全サンプル PASS

**重要な発見**: MOVi-Aではシーンレベルでの metal_only/rubber_only はほとんど存在しない。
大半のシーンが metal+rubber 混合。→ **per-object ARI が唯一の意味ある材質分析**。

### 7.2 正しいラベルでの3 Backbone再評価結果

`src/compute_ari_v2.py` を新規作成。per-object material ARI を計算。
300サンプル（旧60サンプルとは独立した新規データ）での推論のみ評価（再訓練なし）。

#### Table A: Backbone比較（全体）

| Model | Loss | FG-ARI (↑) | std | Full-ARI |
|-------|------|-----------|-----|----------|
| **DINOv2 ViT-S/14** | MSE | **0.168** | 0.183 | 0.005 |
| DINOv1 ViT-S/16 | ch-norm | 0.176 | 0.184 | 0.030 |
| CLIP ViT-B/16 | MSE+det. | 0.047 | 0.103 | -0.022 |

**旧結果との比較**: DINOv2 0.165→0.168, DINOv1 0.153→0.176, CLIP 0.041→0.047
（旧60サンプルのランダムラベルは結果に影響しなかった。全体FG-ARIはラベルに依存しないため妥当）

**注目**: DINOv1が300サンプルではDINOv2を上回った（0.176 > 0.168）。
旧60サンプルでは逆（0.153 < 0.165）。サンプルサイズの増加で逆転が生じた。

#### Table B: Per-Object Material ARI（★新規・核心的結果）

| Model | Metal obj ARI | Rubber obj ARI | Delta (M-R) | 解釈 |
|-------|-------------|---------------|-------------|------|
| **DINOv2** | 0.185 | **0.218** | **-0.032** | **Rubberの方が分離しやすい** |
| DINOv1 | 0.214 | 0.222 | -0.008 | ほぼ均一 |
| CLIP | 0.071 | 0.068 | +0.004 | 両方低い(差なし) |

**★核心的知見**: 
- DINOv2で **Metal obj ARI (0.185) < Rubber obj ARI (0.218)**、差 -0.032
- **正しいラベルで Metal < Rubber が確認された** → 鏡面反射物体は実際に分離が困難
- 旧データでは「Metal > Non-metal」と逆の結果だったが、あれは**ランダムラベルのノイズ**
- DINOv1は差がほぼゼロ（-0.008）→ 材質に依存しない均一な分離
- CLIP は両方低く、材質以前に空間弁別能力が不足

#### Table C: シーン分類別 FG-ARI

| Model | metal_only (n=8) | rubber_only (n=5) | mixed (n=287) |
|-------|-----------------|-------------------|---------------|
| DINOv2 | 0.131 | **0.220** | 0.168 |
| DINOv1 | 0.205 | 0.234 | 0.175 |
| CLIP | 0.015 | 0.022 | 0.048 |

サンプル数が少ないため参考値だが、DINOv2の metal_only (0.131) < rubber_only (0.220) は
per-object結果と整合的。

### 7.3 旧Table 2との対比

| | 旧Table 2（ランダムラベル, 60サンプル） | 新Table B（正しいラベル, 300サンプル） |
|---|---|---|
| DINOv2 Metal vs Non-metal | 0.185 > 0.146 (+0.039) | 0.185 < 0.218 (**-0.032**) |
| DINOv1 | 0.154 ≈ 0.152 (+0.002) | 0.214 ≈ 0.222 (-0.008) |
| CLIP | 0.048 > 0.034 (+0.014) | 0.071 ≈ 0.068 (+0.004) |
| 結論 | 「Metalの方が分離しやすい」（**偽**） | 「Metalの方が分離しにくい」（**真**） |

**旧Table 2は完全に逆の結論**を示していた。ランダムラベルによる偽陽性。

### 7.4 考察と方針決定

**Case A が確認された**: Metal ARI < Rubber ARI（DINOv2で -0.032）

1. **当初の仮説「鏡面反射が困難を引き起こす」は正しかった**。ただしラベルの問題で検証できていなかっただけ
2. **DINOv1が材質差なし** → DINOv1の空間弁別能力（Spatial/Channel比 0.740）が材質非依存の分離に寄与
3. **RQは統合可能**: 「ViT事前学習の違いは、(a) 全体的な分離性能と (b) 材質依存性の両方に影響」

**ポスターの方針**:
- **方向A（材質検証）**: ✅ 成功。Metal < Rubber を正しく示せる
- **方向B（Backbone比較）**: ✅ DINOv2/DINOv1/CLIPの序列は維持（CLIPは低い）。
  ただしDINOv1≧DINOv2 は新発見。300サンプルでの新結果として報告
